Semana 9: Next.js CSR / SSR

Definiciones

Next.js es un framework de desarrollo web basado en React que simplifica y agiliza la creación de aplicaciones web.

Proporciona un conjunto de utilidades y convenciones para la creación de rutas, gestión del estado y la posibilidad de renderizar páginas tanto en el lado del servidor como en el lado del cliente.

Dentro de su uso y utilidad están:

  • Interactividad Rápida: Permite una carga inicial rápida, enviando solo el esqueleto de la aplicación al cliente y realizando la renderización en el navegador.
  • Aplicaciones de Una Página (SPA): Adecuado para aplicaciones con una interactividad intensiva en el cliente, funcionando como SPAs donde las transiciones entre páginas se gestionan sin recargar la página completa.

Procedimientos

Next.js utiliza una serie de procedimientos y conceptos para su ejecución. Aquí una visión de los procedimientos clave:

  • Configuración: Se debe configurar un proyecto Next.js utilizando la herramienta de línea de comandos create-next-app.
  • Ejecutar el Proyecto: Luego se ejecuta el comando: npm run dev.
  • Enrutamiento: Next.js utiliza un sistema de enrutamiento basado en archivos. Los archivos en la carpeta pages determinan las rutas de la aplicación.
  • Datos: Para la creación de páginas dinámicas y estáticas, se utilizan funciones especiales como getStaticProps y getServerSideProps para controlar la obtención de datos.
  • Estilos: Se pueden gestionar los estilos de los componentes utilizando módulos CSS, Styled JSX, o integrando con bibliotecas como SCSS o Tailwind CSS.
  • API Routes: Next.js permite la creación de API Routes para gestionar solicitudes y respuestas en el servidor. Estos archivos se encuentran en la carpeta pages/api.
  • Despliegue: Para desplegar la aplicación, se pueden utilizar diferentes servicios, como Vercel, que tiene una integración estrecha con Next.js, o otros servicios como AWS y Netlify.

Resultados

El uso de Next.js puede proporcionar una experiencia de desarrollo más eficiente, mejorar el rendimiento y facilitar la implementación de estrategias avanzadas, como la representación del lado del servidor y el SEO mejorado.

La configuración inicial de un proyecto Next.js es rápida y sencilla utilizando create-next-app, con recarga en tiempo real y actualización automática durante el desarrollo local, facilitando un desarrollo rápido y eficiente.

La creación de API Routes en la carpeta pages/api simplifica la gestión de solicitudes y respuestas en el servidor.
